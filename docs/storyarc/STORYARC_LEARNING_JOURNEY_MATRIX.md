# StoryArc Learning Journey Matrix

One learning graph across three tracks. Learning model: UNDERSTAND → EXPERIENCE → USE → PRACTICE → PROVE → RECALL.

- **SOURCE LEARNING ITEM** — School Core item introducing the objective (UNDERSTAND/USE, includes an active production task).
- **PRACTICE** — formative practice location(s): School Core drills and Story Mode LEARNING PRACTICE beats.
- **STORY APPLICATION** — episode(s) where the objective is applied with narrative consequences (EXPERIENCE).
- **RECALL EVENT** — later, delayed retrieval point (RECALL; evidence-eligible when rubric-declared). Depends on gap G1 (recall scheduler) for out-of-story spaced review; in-story recall scenes work without it.
- **ASSESSMENT EVIDENCE** — Exam Lab item(s) producing assessed evidence (PROVE).

Objective keys are stable (`obj-g10-01`…); display wording lives in published content. Item IDs are defined in `STORYARC_CONTENT_MANIFEST.md`. Skills: L listening, S speaking, R reading, W writing, G grammar, V vocabulary.

## Graph invariants (release validation targets)

1. Every objective has exactly one source item, ≥1 story application, ≥1 assessment evidence link; every objective except obj-g12-15 has ≥1 recall event.
2. Every Story episode applies ≥2 same-grade School Core items; every episode except sm-g10-01 contains ≥1 recall reference.
3. ≥80% of Exam Lab scored items trace to an objective in this matrix (blueprint enforces per item).
4. Cross-grade recall chains (must survive content revisions): obj-g10-12 → sm-g12-05; obj-g10-15 → sm-g12-09; obj-g10-05/04 → sm-g11-08; obj-g11-08 → sm-g12-05; obj-g10-14 → sm-g11-02 → sm-g12-02/03.
5. Expression-level recall links are governed by `STORYARC_VOCABULARY_EXPRESSION_STRATEGY.md`.

CONTENT REQUIREMENT / IMPLEMENTATION GAP: out-of-story spaced recall (review prompts between sessions) requires the recall scheduler (gap G1). In-story recall scenes above do not depend on it.
